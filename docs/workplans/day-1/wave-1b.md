# D1-WAVE-1B — Portfolio and Data Workspace

Objective: define and implement reviewed local CSV/YAML preview, validation,
confirmation, immutable snapshots/comparison, source/provider catalogue, and
fixed query-manifest surfaces.

Dependency: accepted 1A contracts. Participants: data, experience,
integration. External providers, broker connections, licensed/personal Git
data, arbitrary SQL, and mutation of existing snapshots are excluded. Provider
defaults must remain disabled and all quality, rights, access, provenance,
freshness, data-zone, and publication states must be visible. Acceptance is
the 1B gate in `config/agent/day1/waves.json`.
