# D1-KP-01: Human Workbench and Information Architecture

Status: review requested. Deadlines: draft T1+2h; review T1+4h.

The Workbench primary experience is server-rendered semantic HTML with local CSS and progressive enhancement. Dashboard, Portfolio, Risk, Findings, Alerts, Data, Providers, Research, Notebooks, Agents, Plan, and Settings remain distinct destinations. JSON remains available as a developer and evidence API, not the primary presentation.

Every screen exposes the active `research` or `personal_portfolio` profile, public/private-local/synthetic evidence state, freshness or missingness, and human-review state. Heading order, labelled controls, keyboard focus, responsive layout, readable values, and explicit empty and error states are required. Missing data is never rendered as zero.

Research and notebook screens present reviewed catalogue metadata only. They offer no execution control. This planning record does not modify the application and does not authorize provider access, external assets, arbitrary execution, or consequential action.
